Getting started

A guide for your first steps with Daisy

Daisy works alongside jQuery Mobile version 1.4 and above. If you want to use Daisy properly you must follow official jQuery Mobile code rules which are very, very, very, very simple.

Daisy is built on jQuery Mobile and supports majority of mobile devices.

Useful stuff

You could find useful list of examples that comes with the template. Also you can use official jQuery Mobile Docs and Demos page.

  • jQuery Mobile Official Demos
  • jQuery Mobile Docs
  • Documentation
  • Support forum for Daisy

Usual stuff about site building

There are few rules you need to follow in order to keep all the functionalities.

  1. Always check in <head> if there are all nerccessary libraries
  2. It's advised to follow basic jQM structure (see below)
  3. If something doesn't work check step 1 and 2
  4. If step 3. doesn't do the trick send me an email

HTML Head

Mobile devices markup

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />

Import jQuery Mobile CSS

<link rel="stylesheet" href="css/jqeury.mobile.theme.min.css" />
<link rel="stylesheet" href="css/jquery.mobile.icons.min.css" />
<link rel="stylesheet" href="css/jquery.mobile.min.css" />

Import CSS styles

<link rel="stylesheet" href="css/style.css" />

Import nercessary style libraries

<link rel='stylesheet' href='css/fonts.css' type='text/css'>
<link rel='stylesheet' href='css/luke.css' type='text/css'>

Load Font Awesome icons

<link rel="stylesheet" href="css/font-awesome.min.css">

Load jQuery (JS)

<script src="js/jquery.min.js"></script>

Load jQuery Mobile (JS)

<script src="js/jquery.mobile.min.js"></script>

HTML Page Structure

You can create your own HTML page using this basic structure.

<body>
  <div data-role="page">
    <div data-role="header">
    </div>
    
    <div data-role="content">  
    </div>   
    
    <div data-role="footer">  
    </div>
  </div>
</body>
