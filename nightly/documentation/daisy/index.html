HTML

Page structure

Both templates work with jQuery Mobile version 1.4 and above. If you want to use this template with all it's functions and features you will have to follow simple HTML layout.

Keep in mind that there are certain libraries that must be included within <head> selection. Basic header that can be used with all necessary plugins looks like this:

<head>
        <title>[page title]</title>
        <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />
 
        <!-- jQueryMobileCSS - styling-->
        <link rel="stylesheet" href="css/jqeury.mobile.theme.min.css" />
 
        <!-- jQueryMobile icons -->
        <link rel="stylesheet" href="css/jquery.mobile.icons.min.css" />
 
        <!-- jQueryMobile core CSS -->
        <link rel="stylesheet" href="css/jquery.mobile.min.css" />
       
        <!-- Fonts -->
        <link rel="stylesheet" href="css/fonts.css" />
        
        <!-- Main CSS styles -->
        <link rel="stylesheet" href="css/style.css" />
        
        <!-- Luke CSS styles -->
        <link rel="stylesheet" href="css/luke.css" />
 
        <!-- Load Font Awesome Icons -->
        <link rel="stylesheet" href="css/font-awesome.min.css">
 
        <!-- jQuery -->
        <script src="js/jquery.min.js"></script>
        
        <!-- jQuery Mobile Scripts -->
        <script src="js/jquery.mobile.min.js"></script>
    		
        <!-- Main Scripts -->
        <script src="js/scripts.js"></script>
 
</head>

jQuery Mobile

jQuery Mobile is a HTML5-based user interface system designed to make responsive web sites and apps that are accessible on all smartphone, tablet and desktop devices.

The jQuery Mobile serves as the core of the template providing all the great jQM features (like grids, responsiveness, mobile browsers support, libraries, functions, and much more) with the template on the top providing the brilliant GUI and exquisite user experience.

Data Variables

The jQuery Mobile framework uses HTML5 data- attributes to allow for markup-based initialization and configuration of widgets.

Nightly and Daisy inherited data- variables and uses them quite extensively. Proper usage of data variables can be found in jQuery Mobile Docs. However it's very simple and saves you a lot of work. Proper use of data variables looks like this:

<button data-role="button" data-theme="a" data-corners="false">Test button</button>

There are many data variables available. Worry not, you will probably never use 90% of them. But there are few worth remembering as they are used very often in the template:

data-role [role]     (apply jQuery Mobile native role)
data-theme[theme-code (a/b/c/d/e)]        (changes color scheme)
data-ajax[true/false]    (turns on/off AJAX page loading)
data-border[true/false]  (adds/removes border from element)
data-corners[false] (removes rounded corners)
data-shadow[false]  (removes shadows)
data-inner[true/false]  (true: adds extra padding and margin / false: adds margin only)

jQuery Mobile official article covering this subject:

http://api.jquerymobile.com/data-attribute/

luke.css

Styling

Library luke.css contains all sort of useful pre-coded elements such as forms, inputs, tables, typography and you can find them used through the whole template. The usage is really simple as is the additional styling.

For using one of luke.css elements it's important to switch off the jQuery Mobile for the element. This is due to jQuery Mobile own styling and we don't want that. Calling data-role="none" disables jQuery Mobile for any given element.

Use luke.css classes for creating requested element for example button. Class for button is .luke-button (you can notice that all luke.css classes starts with luke- Final code looks like this:

<button data-role="none" class="luke-button">Luke button</>

Now the result would be plan button and that is perfectly okay. Additional color themes can be used, same color themes available for the whole template. You can change color theme using data-theme property (same as in jQuery Mobile). If we want to create button with the color theme "a" we would use the code below:

<button data-role="none" class="luke-button" data-theme="a">Luke button</>

It is very, very, very simple as you can see. I was even considering if I should even write this but then I said nah I'll be fun so here we are.

Same approach can be used for any luke.css element (whole list below).

Customizing theme

Color themes

A theme (in jQuery Docs it's called swatch) is one of several color schemes provided by your theme. Template uses single-letter designations for themes (swatches). The default jQuery Mobile theme provides two swatches. In this template you will find three more.

List of themes:

  1. "a" - Blue theme
  2. "b" - Dark theme
  3. "c" - Rather subtle blue
  4. "d" - White theme
  5. "e" - Yellow theme

The ThemeRoller allows you to create any number of swatches for a custom theme.

Using own colors

If you want to add/change color themes (swatches) used in the theme there is very simple way to do so.

1. step ThemeRoller

Open ThemeRoller and import current themes library file css/jquery.mobile.theme.min.css after import you will see all themes available as in the theme. Now you can change/add/delete/adjust color variations for chosen themes. After finishing your work simply export your new themes library and overwrite original file.

2. step CSS changes

Now after you have imported ThemeRoller file into the template you need to change color variations in the style.css file as well. You see ThemeRoller works with jQuery Mobile framework and don't affect any other part of the template created separately (lile luke.css elements for example).

Open css/style.css and find any data-theme entry and then change it according your color variations from ThemeRoller. For example:

[data-role=panel][data-theme=a]{
    background: #181b22;
    border-right: 1px solid #181b22;
    color: #fff;
}

Same approach use with file css/luke.css Now here just scroll down the file where you will find all theme variations listed at once just like this:

.ui-page[data-theme=a], .luke-text-box[data-theme=a], .ui-tag[data-theme=a], .luke-bg[data-theme=a], .luke-cute-underline[data-theme=a]::after, .luke-box[data-theme=a], .luke-circle[data-theme=a], .luke-button[data-theme=a], .luke-input[data-theme=a], .ui-footer[data-theme=a] [data-role=navbar] ul li a, .highlight[data-theme=a] {
    background: #181b22;
    border-color: #181b22 !important;
    color: #fff;
}

Change colors, hit save and that's it.
Remember:

  1. First import/create/edit themes in ThemeRoller, export and replace file in the CSS folder
  2. Then change colors in style.css
  3. And then change colors in luke.css as well

Swipe

Swipe feature is available for opening/closing sidebars. The initial swipe is set to the sidebar with ID#sidebar what that means is that if you gave your sidebar ID #sidebar it will respond on swipe.

Now there are two things you would want to do with this. Either turn off the swipe feature or add another swipe ID.

I will now show you the swipe handler with descriptions for each line. You can find similar code in the scripts.js file. You don't need to be jQuery guru for editing this code because it's very simple however if you want to know how this works feel free to ask me.

// Sidebar swipe opening/closing
jQuery( document ).on( "swipeleft swiperight", function( e ) { // when you swipe your finger
	if ( $.mobile.activePage.jqmData( "panel" ) !== "open" ) { // and if panel isn't already open
  	if ( e.type === "swipeleft"  ) { // and if the swipe is from right to left
  		jQuery( "#sidebar-right" ).panel( "open" ); // open right sidebar with ID #right-sidebar
    } else if ( e.type === "swiperight" ) { // if the swipe is from left to right
    	jQuery( "#sidebar" ).panel( "open" ); // open (left) sidebar with ID #sidebar
  	} 
  	
 	}
 });

Now you probably know if you want to use your own swipe ready sidebar ID you can add another line for example jQuery( "#my-awesome-sidebar" ).panel( "open" ); but I suggest you just use IDs given by default (#sidebar and #right-sidebar).

If you want to turn off the swipe feature open file js/scripts.js and work with the same code as above.

// Sidebar swipe opening/closing

Destroy it! Delete the whole thing.

Disable swipe caller on certain elements (for example sliders). Add class .disableswipe to the element.

Device Detection

Is lightweight jQuery script located in js/device.detect.js which can detect which device are you using. You can work whit it as a library and it's free to use. It can detect Android, BlackBerry, iOS and last but not least Microsoft's IE. Furthermore it can tell you which Apple device are you using (iPhone, iPad or iPod).

Importing js library in page header gives you access to following functions:

deviceDetect.Android(); // returns true if Android
deviceDetect.BlackBerry(); // returns true if BlackBerry
deviceDetect.iOS(); // returns true if iOS device
deviceDetect.any(); // returns true if mobile device
isRetina(); // returns true if Retina
$(window).width(); // returns window width
$(window).height(); // returns window height
window.screen.width; // returns actual screen width
window.screen.height; // returns actual screen height

GPS Tracking

Both templates comes with great feature that will make your web applications stand out! The feature is location tracking via browser for computers or GPS location tracking for mobiles. Browser's location (it will probably ask you to accept or deny the request) provides information about the latitude and longitude. Then the AJAX call to Google API retrieves the address for our geolocation.

This library uses Google API so for more detailed docs use Google official help. See page location.html for further information. List of basic functions:

address_components[2].short_name - street name
address_components[1].short_name - house number
address_components[0].short_name - house number
address_components[3].short_name - city
address_components[7].short_name - state
lat - GPS latitude
lng - GPS longtitude

Making APPS with Nightly+Daisy

jQuery Mobile provides compatibility with PhoneGap as well as Cordova.

PhoneGap is a free and open source framework that allows you to create mobile apps using standardized web APIs for the platforms you care about.
Apache Cordova is a set of device APIs that allow a mobile app developer to access native device function such as the camera or accelerometer from JavaScript. Combined with a UI framework such as jQuery Mobile or Dojo Mobile or Sencha Touch, this allows a smartphone app to be developed with just HTML, CSS, and JavaScript.

If you are interested in creating applications with Nightly or Daisy read PhoneGap or Cordova official website and learn more.
